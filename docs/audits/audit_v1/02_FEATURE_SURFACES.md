# Feature Surfaces Inventory

## Routing Model
- Client routing is query-param based in `src/App.tsx` (`view` param) rather than React Router.
- Main views: `dashboard`, `streak-dashboard`, `tracker` (grid/day toggle), `goals`, `wins`, `routines`, `journal`, `tasks`, `wellbeing-history`, `debug-entries`.

## iPhone / PWA Interaction Gotchas
| Area | Evidence | Impact |
| --- | --- | --- |
| Tap interaction assumptions | Double-click delete pattern in tracker (`src/components/TrackerGrid.tsx:1083-1186`) | Touch devices lack reliable dblclick; accidental/blocked delete actions. |
| Modal accessibility/focus | Many modal overlays are div-based without `role="dialog"`/`aria-modal`/focus trap patterns (example: `src/components/RoutineRunnerModal.tsx:126-147`) | Keyboard + screen reader friction, especially on iPad + hardware keyboard. |
| Sticky/fixed layout stacking | Fixed header (`src/components/Layout.tsx:99`) + nested sticky grid header (`src/components/TrackerGrid.tsx:1202`) + multiple scroll containers | Mobile viewport jumps, clipped content, and hit-target overlap risk. |
| PWA icon/install polish | Manifest uses `vite.svg` for both 192/512 icons (`public/manifest.webmanifest:11-21`) | Poor install icon quality on iOS/Android homescreens. |
| Offline behavior expectations | Service worker excludes `/api/*` from cache (`public/sw.js:29`) and has no mutation queue | App shell works offline, but logging/actions fail hard without network. |
